# Tythan Code — полное руководство (RU)

## Суть продукта в двух абзацах

Tythan Code — это расширение VS Code, которое даёт рабочий процесс Cursor
(чат-агент с доступом к файлам, мультифайловый Composer, inline-правки,
автодополнение, @codebase) поверх **обычного** VS Code, с двумя отличиями,
которые и есть наше позиционирование:

1. **Security-first**: встроенный офлайн-сканер уязвимостей (секреты в коде,
   SQL-инъекции, отключённый TLS, слабая криптография, опасные конфиги) —
   агент обязан прогонять его после правок; плюс аудит всего проекта одной
   командой.
2. **Любая модель и полный контроль**: Anthropic напрямую или любой
   OpenAI-совместимый эндпоинт — включая локальные Ollama/LM Studio/vLLM,
   когда код нельзя выносить за периметр. Ключи в системном SecretStorage,
   индексация кода — офлайн, каждая правка — через подтверждаемый дифф с undo.

## Как это работает внутри (архитектура)

```
Пользователь ⇄ Чат-webview / команды VS Code (vscode-integration/)
                    │
                    ▼
              Agent (core/agent.ts) — цикл:
                модель → tool_use → выполнение → tool_result → модель…
                    │                       │
        Backend (providers/)         Workspace (tools.ts)
        Anthropic | OpenAI-совм.     read/write/edit/search/run — всё
        стриминг, отмена, usage      заперто внутри корня проекта
                    │                       │
             Compaction (сжатие      Checkpoints (undo) /
             старых ходов при        OverlayWorkspace (Composer:
             переполнении контекста) правки в памяти до review)
```

Ключевые механики:

- **Агентский цикл.** Модель получает системный промпт (+ правила из
  `.tythanrules`/`.cursorrules`/`AGENTS.md`) и 7 инструментов: `read_file`,
  `write_file`, `edit_file`, `list_files`, `search`, `security_scan`,
  `run_command`. Мутации требуют подтверждения диффом (или YOLO-режим).
- **Чекпоинты.** Перед каждой правкой файла сохраняется его состояние;
  один ход = один чекпоинт; «Undo Last Agent Change» откатывает весь ход.
- **Компакция контекста.** При приближении к пределу окна старые ходы
  сжимаются в конспект — длинная сессия не падает с ошибкой контекста.
- **Composer.** Тот же агент, но пишет в OverlayWorkspace — виртуальный
  слой поверх диска. Агент видит свои правки (может перечитывать и
  дорабатывать), диск не тронут; в конце — review-список с чекбоксами.
- **@codebase.** Локальный TF-IDF-поиск по чанкам кода с разбиением
  идентификаторов (`getUserProfile` найдётся по запросу «user profile»).
- **Сессии.** История чата (и «сырая» история модели) сохраняется на диск
  по workspace'у; переключение между сессиями — `Chat Sessions…`.

## Установка и запуск

### Вариант А: готовый пакет

```bash
code --install-extension tythan-code-0.4.0.vsix
```

### Вариант B: из исходников

```bash
cd tythan-code
npm install
npm run build        # esbuild → dist/extension.js
npm run package      # → tythan-code-0.4.0.vsix
code --install-extension tythan-code-0.4.0.vsix
```

Для разработки: откройте папку `tythan-code/` в VS Code и нажмите `F5` —
поднимется Extension Development Host с расширением.

### Первичная настройка

1. Откройте любую папку-проект (расширению нужен workspace).
2. `Ctrl+Shift+P` → **Tythan Code: Set API Key for Provider** → вставьте ключ
   (хранится в SecretStorage, не в settings.json). Переменные окружения
   `ANTHROPIC_API_KEY` / `OPENAI_API_KEY` / `OPENROUTER_API_KEY`
   подхватываются автоматически.
3. Провайдер/модель: настройки `tythanCode.provider` / `tythanCode.model`
   или команда **Select Model**. Для локальной модели: провайдер `ollama`
   (или `custom` + `customBaseUrl`).

## Все функции и как ими пользоваться

| Функция | Как вызвать | Что происходит |
|---|---|---|
| Чат-агент | Иконка в панели / `Ctrl+Alt+;` | Стриминг, инструменты, диффы на правки, код-блоки с Copy/Insert, кнопка Stop |
| Composer | `Ctrl+Alt+I` | Мультифайловая задача → staged-правки → review-список с чекбоксами → один чекпоинт |
| Inline-правка | Выделить код → `Ctrl+Alt+K` | Описываете изменение → дифф всего файла → Apply (обычный Ctrl+Z откатывает) |
| Выделение в чат | `Ctrl+Alt+L` | Код с путём и строками попадает в поле чата |
| Fix Problems | `Ctrl+Alt+.` или 💡 на ошибке | Диагностики файла уходят агенту, он чинит через диффы |
| @файл / @codebase | В тексте сообщения | Прикрепить файл целиком / подтянуть релевантные куски кода |
| Security-аудит | Команда **Run Security Audit** | Офлайн-отчёт по секретам/уязвимостям/конфигам |
| Commit message | ✨ в панели Source Control | Сообщение по diff'у сразу в поле коммита |
| Сессии | **Chat Sessions…** | Переключение/удаление сохранённых диалогов |
| Undo | **Undo Last Agent Change** | Откат всех файловых правок последнего хода |
| Правила | `.tythanrules` в корне | Инструкции проекта в каждом системном промпте |
| Автокомплит | Просто печатайте | Ghost-text; ускоряется настройкой `inlineCompletion.model` (быстрая модель) |

## Тонкая настройка под себя

- `tythanCode.effort` — глубина размышлений (Anthropic): `low` для скорости,
  `max` для сложных задач.
- `tythanCode.inlineCompletion.model` — маленькая модель для автокомплита
  (например `claude-haiku-4-5-20251001`), чат остаётся на большой.
- `tythanCode.yolo` — автоприменение правок без подтверждений. Опасно,
  но удобно на одноразовых проектах. Переключается командой Toggle YOLO.
- `tythanCode.contextWindow` — задать явно для локальных моделей.

## Проверка качества (что уже сделано)

- `npm run typecheck` — строгий TypeScript по всем файлам.
- `npm test` — 191 офлайн-тест: цикл агента, оба бэкенда, инструменты
  и конфайнмент путей, чекпоинты/undo, компакция, staging Composer'а,
  ретривер @codebase, rules, сессии, security-сканер.
- `npm run test:integration` — smoke-тест в настоящем VS Code
  (нужна машина с обычным интернетом; в облачной песочнице недоступен).

## Известные ограничения (говорим честно)

- Автокомплит — не специализированный FIM-эндпоинт: ~секунда на подсказку
  (кэш и быстрая модель сильно помогают, но Copilot-скорости нет).
- `run_command` не покрывается undo (откатить произвольную команду нельзя).
- SCA-скан зависимостей (CVE по lock-файлам) пока не портирован из CLI.
- Живой UI-тест в реальном VS Code запускайте у себя (`F5`) — облачная
  среда сборки не имеет бинарника VS Code.
