# Tythan Code — путь к продажам (Go-to-Market, RU)

Честная рамка: Cursor шёл к сотням миллионов долларов ARR несколько лет,
с венчурными деньгами, собственным форком VS Code и своими моделями.
Догнать их «в лоб» нельзя — и не нужно. Реалистичная цель первого года:
**нишевый продукт с первыми сотнями платящих** на позиционировании, которое
Cursor закрыть не может. Дальше — рост от ниши.

## 1. Позиционирование: почему купят у нас, а не Cursor

**«AI-кодинг для тех, кому нельзя утекать».** Три аргумента:

1. **Security-first.** Встроенный аудит уязвимостей и секретов, агент
   обязан сканировать свои правки. У Cursor этого нет из коробки.
2. **Суверенность данных.** Работает с локальными моделями (Ollama/vLLM) и
   любым провайдером; индексация кода офлайн; ключи в SecretStorage.
   Продаётся банкам, госсектору, финтеху, всем с NDA-кодом.
3. **Не форк.** Это расширение: живёт в обычном VS Code клиента, без смены
   IDE, без ожидания рибейзов форка, совместимо с их политиками.

Целевая аудитория по порядку приоритета: (а) командные лиды/CTO компаний
50–500 человек с требованиями безопасности; (б) секьюрити-инженеры;
(в) индивидуальные разработчики, которым нужен «Cursor со своим ключом»
(они платят провайдеру напрямую, нам — за продукт).

## 2. Модель монетизации

| Тир | Цена | Что входит |
|---|---|---|
| Free | 0 | Всё текущее, свой API-ключ (BYOK). Двигатель дистрибуции |
| Pro | ~$10/мес с разработчика | SCA-скан зависимостей (CVE), приоритетные фичи, поддержка |
| Team | ~$25/мес с разработчика | Центральные политики (правила, запрет YOLO), общие rules, аудит-лог правок агента, SSO |

Принцип: **инференс оплачивает клиент своим ключом** — у нас нет затрат на
GPU и нет маржинальной ямы, в которой сидят конкуренты. Мы продаём
инструмент и контроль, не токены.

## 3. Дорожная карта до денег (12 месяцев)

**Месяц 1–2 — фундамент доверия.**
- Опубликовать в VS Code Marketplace + Open VSX (это бесплатно).
- Довести репозиторий до образцового вида: лицензия, CI на GitHub Actions
  (typecheck + 191 тест уже есть), CHANGELOG, GIF-демо в README.
- Прогнать демо-сценарий (`demo/README.ru.md`) на 10 живых разработчиках,
  зафиксировать, где спотыкаются.

**Месяц 3–4 — дистрибуция.**
- Запуски: Product Hunt, Hacker News (Show HN), r/LocalLLaMA (наш козырь —
  локальные модели), Habr для русскоязычного рынка.
- Контент: «Мы просканировали N популярных репозиториев — вот что нашёл
  наш бесплатный сканер» — такие посты дают органический трафик.
- Метрика цели: 2–5 тыс. установок, 300+ недельно активных.

**Месяц 5–8 — первые деньги.**
- Выпустить Pro (SCA + поддержка) с простым лицензионным ключом.
- 20 «design partner»-компаний: бесплатный Team-тир в обмен на фидбек и
  логотип. Из них — первые 5 платящих команд.
- Цель: $2–5 тыс. MRR. Это мало, но это доказательство продукта.

**Месяц 9–12 — команды.**
- Team-тир: центральные политики, аудит-лог, SSO. Продажа руками:
  секьюрити-лидам, через кейсы design-партнёров.
- Цель к концу года: $10–30 тыс. MRR. Дальше видно, поднимать ли раунд.

## 4. Что дострочить в продукте перед стартом продаж

1. SCA-скан зависимостей (OSV.dev по lock-файлам) — ядро Pro-тира.
2. GIF/видео-демо 60 секунд — конверсия Marketplace-страницы решается им.
3. Телеметрия opt-in (только счётчики функций) — иначе не увидите, что
   используют.
4. Публикация: аккаунт издателя `tythanai` в Marketplace, подпись vsix.
5. Лендинг с одной кнопкой «Install» и одним видео.

## 5. Чего НЕ делать

- Не соревноваться с Cursor в скорости автокомплита — это гонка моделей,
  она не наша. Наша гонка — доверие и контроль.
- Не строить свой инференс/подписку на токены на старте — сожжёт кассу.
- Не пилить форк VS Code — похоронит команду в мейнтенансе.
- Не добавлять фичи до публикации в Marketplace: дистрибуция > фичи.

## 6. Метрики, за которыми следить

Установки → недельно активные → «активировавшиеся» (первый принятый дифф
агента) → удержание 4-й недели → конверсия в Pro. Северная звезда:
**принятые правки агента в неделю** — она предсказывает и удержание, и
готовность платить.
