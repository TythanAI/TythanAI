# 🛒 Telegram-бот-магазин (автовыдача аккаунтов и цифровых товаров)

Бот продаёт цифровые товары прямо в Telegram и **выдаёт их автоматически** после
оплаты: покупатель выбирает товар, платит — и сразу получает его в чат.

> ⚠️ **Важно.** Перепродажа Telegram-аккаунтов нарушает
> [Условия использования Telegram](https://telegram.org/tos) — такой бот и аккаунт
> продавца могут заблокировать. Технически это универсальный магазин цифровых
> товаров: что грузить в каталог (аккаунты, ключи, подписки, файлы) — решаешь ты,
> и ответственность за это тоже на тебе.

---

## Возможности

**Для покупателя:**
- 🛍 Каталог с **категориями и подкатегориями**
- 🛒 Покупка в один клик и **мгновенная автоматическая выдача**
- 🎟 **Промокоды** (скидка % или фиксированная)
- 🧾 История покупок · ⭐️ **отзывы** · 💬 поддержка

**Для тебя (админка, видит только владелец):**
- ➕ Товары (мастер), 📦 загрузка «наличия» пачкой (по строке на единицу)
- 🗂 Категории и подкатегории
- 🎟 Промокоды (создать/удалить)
- 📣 **Рассылка** всем пользователям
- ⭐️ Просмотр отзывов, 📊 статистика, 🔔 уведомления о продажах
- 💾 **Бэкапы базы** в Telegram-чат (по расписанию и по кнопке)

**Оплата — на выбор, без сайта и без кода:**
- 💳 **Карта/крипта вручную** (`manual`) — покупатель платит по твоим реквизитам,
  ты подтверждаешь → бот сам выдаёт товар. Ничего подключать не надо.
- 💎 **Криптовалюта TON** — включается отдельно (`TON_WALLET` + `TON_RATE`) и
  показывается покупателю как ещё одна кнопка. Бот **сам проверяет перевод по
  блокчейну** (toncenter) и выдаёт товар автоматически.
- ⭐️ **Telegram Stars** (`stars`) — оплата звёздами внутри Telegram.
- 🏦 **Провайдер** (`provider`) — карты через платёжного провайдера (нужен токен).

Защита от ошибок: одну и ту же единицу нельзя продать дважды; при ручной/TON
оплате товар резервируется за покупателем; если для звёзд наличие кончилось в
момент оплаты — звёзды возвращаются автоматически.

## ⚙️ Как всё подключить — по шагам (10 минут)

### Шаг 1. Создай бота и получи токен
1. Открой [@BotFather](https://t.me/BotFather) → `/newbot`.
2. Придумай имя и username (заканчивается на `bot`).
3. Скопируй **токен** вида `123456789:AAE...`.

### Шаг 2. Узнай свой Telegram ID
Открой [@userinfobot](https://t.me/userinfobot) → **Start** → он пришлёт `Id: 123456789`.

### Шаг 3. Установи зависимости
Нужен **Python 3.10+** (`python --version`).
```bash
cd telegram_shop_bot
pip install -r requirements.txt
```

### Шаг 4. Настрой `.env`
```bash
cp .env.example .env
```
Открой `.env` и заполни:
```dotenv
BOT_TOKEN=токен_из_шага_1
ADMIN_IDS=твой_ID_из_шага_2
SHOP_NAME=Мой магазин

# Приём рублей на карту (рекомендуется):
PAYMENT_METHOD=manual
CURRENCY=RUB
PAYMENT_DETAILS=Сбербанк 2202 2000 0000 0000 (Иван И.)
```
> Хочешь принимать **звёздами** (совсем без реквизитов) — поставь
> `PAYMENT_METHOD=stars`. Хочешь **карты автоматически** — `PAYMENT_METHOD=provider`
> и укажи `PROVIDER_TOKEN` из @BotFather (Bot Settings → Payments).

### Шаг 5. Запусти
```bash
python run.py
```
Увидел `Бот @… запущен` — готово! Открывай бота, жми **Start**.
> Пока открыто это окно — бот работает. Для работы 24/7 см. раздел «Запуск 24/7».

### Шаг 6. Наполни магазин (как админ)
`/start` → **🛠 Админка**:
1. **🗂 Категории** → создай, например, «Аккаунты», зайди внутрь → добавь «Telegram», «Instagram».
2. **➕ Добавить товар** → название, описание, цена, категория.
3. **📦 Добавить наличие** → выбери товар → пришли единицы **по одной на строку**:
   ```
   login1:pass1
   login2:pass2
   ```
   Каждая строка — то, что бот выдаст очередному покупателю.
4. (по желанию) **🎟 Промокоды**, **📣 Рассылка**.

### Как проходит покупка
Клиент: **🛍 Каталог** → товар → **🛒 Купить**.
- При `stars`/`provider` — оплатил и сразу получил товар.
- При `manual` — видит твои реквизиты, платит, жмёт «✅ Я оплатил», ты в админке
  жмёшь «✅ Подтвердить» → бот сам отправляет товар покупателю.

## 💎 Крипто-оплата TON (по желанию)
Заполни в `.env` две строки — и у покупателя появится кнопка «💎 TON»:
```dotenv
TON_WALLET=EQ...   # адрес твоего TON-кошелька
TON_RATE=500       # сколько единиц цены (₽) в 1 TON: 1 TON = 500 ₽ → 500
```
Бот показывает покупателю адрес и точную сумму с уникальным комментарием, сам
следит за блокчейном и при поступлении оплаты автоматически выдаёт товар. Ключ
`TON_API_KEY` (бесплатно у @tonapibot) не обязателен, но повышает лимиты.

## 💾 Бэкапы базы в Telegram (по желанию)
Создай приватный канал, добавь бота админом, узнай его ID и укажи:
```dotenv
BACKUP_CHAT_ID=-1001234567890
BACKUP_INTERVAL_HOURS=24
```
Бот будет присылать туда копию базы (товары/заказы). Также в админке есть кнопка
**💾 Бэкап базы** — пришлёт копию сразу (если чат не задан — тебе в личку).

## 🖥 Запуск 24/7 (чтобы бот не выключался)

> 📄 Покупаешь VPS? Отдельная подробная инструкция «с нуля» — в файле
> **[DEPLOY.md](DEPLOY.md)** (пошагово: SSH, установка, systemd, обновление).

### Вариант А. Railway (без своего сервера, самый простой)
1. Залей папку `telegram_shop_bot` в репозиторий GitHub.
2. На [railway.app](https://railway.app) → **New Project** → **Deploy from GitHub repo**.
3. В настройках сервиса → **Variables** добавь переменные из своего `.env`
   (`BOT_TOKEN`, `ADMIN_IDS`, `PAYMENT_METHOD`, `CURRENCY`, `PAYMENT_DETAILS`, …).
4. **Start Command**: `python run.py`. Railway сам поставит зависимости.
5. Чтобы база не терялась при перезапуске — подключи **Volume** и смонтируй его в
   папку `data` (Settings → Volumes → Mount path `/app/data`), а в переменных
   задай `DB_PATH=/app/data/shop.db`.

> Аналогично работают Render, Fly.io, любой хостинг с Python/Docker.

### Вариант Б. Свой сервер (VPS) через systemd
Готовый файл — `deploy/shop-bot.service`. Отредактируй пути/пользователя, затем:
```bash
sudo cp deploy/shop-bot.service /etc/systemd/system/shop-bot.service
sudo systemctl daemon-reload
sudo systemctl enable --now shop-bot
systemctl status shop-bot     # проверить
journalctl -u shop-bot -f     # логи
```

### Вариант В. Docker
```bash
docker build -t shop-bot .
docker run -d --name shop-bot --restart=always \
  --env-file .env -v "$PWD/data:/app/data" shop-bot
```
`-v .../data` сохраняет базу между перезапусками.

## ❓ Если что-то не работает

| Проблема | Решение |
|---|---|
| `BOT_TOKEN не задан` | Нет `.env` или пустой токен. Повтори шаги 1 и 4. |
| `ADMIN_IDS не задан` | Впиши свой числовой ID (шаг 2). |
| `PAYMENT_METHOD=manual требует PAYMENT_DETAILS` | Впиши реквизиты карты в `.env`. |
| `PAYMENT_METHOD=provider требует PROVIDER_TOKEN` | Возьми токен в @BotFather или используй `manual`/`stars`. |
| `TON_WALLET задан, но TON_RATE не указан` | Впиши `TON_RATE` — сколько ₽ в 1 TON. |
| TON-оплата не подтверждается | Покупатель должен указать точный комментарий и сумму; проверка идёт через toncenter (10–40 сек). Задай `TON_API_KEY` при частых проверках. |
| Бэкапы не приходят | Бот должен быть **админом** в канале; `BACKUP_CHAT_ID` — с минусом (`-100…`), `BACKUP_INTERVAL_HOURS` > 0. |
| Не вижу «Админка» | Твой ID в `ADMIN_IDS` ≠ аккаунт, из которого пишешь боту. |
| Бот молчит | Проверь, что `python run.py` запущен и без ошибок в консоли. |
| `Unauthorized` | Неверный токен — возьми новый у @BotFather (`/token`). |

## 🗂 Структура кода

```
telegram_shop_bot/
├── run.py                 # запуск бота
├── .env.example           # шаблон настроек
├── requirements.txt       # зависимости
├── Dockerfile             # запуск в Docker
├── deploy/shop-bot.service# systemd-юнит (24/7)
├── bot/
│   ├── config.py          # настройки из .env (способ оплаты, валюта…)
│   ├── database.py        # SQLite: товары, категории, наличие, заказы, промокоды, отзывы
│   ├── texts.py           # ВСЕ тексты бота — меняй тут
│   ├── keyboards.py       # кнопки
│   ├── states.py          # шаги мастеров
│   ├── filters.py         # проверка «это админ?»
│   ├── utils.py           # скидки, промокоды, вспомогательное
│   ├── handlers/          # user.py (покупатель), admin.py (админка), payments.py (оплата)
│   └── services/          # ton.py (TON-оплата), backup.py (бэкапы), orders.py (выдача)
└── tests/                 # тесты (pytest)
```

Меняешь тексты/эмодзи — правь `bot/texts.py`. Технологии:
[aiogram 3](https://docs.aiogram.dev) + SQLite (база создаётся сама в `data/shop.db`).

Тесты: `pip install pytest pytest-asyncio && pytest`.

- `.env` и `data/*.db` **не** попадают в git (см. `.gitignore`) — там токен и заказы.

- Админка доступна только ID из `ADMIN_IDS`; остальные её не видят.
