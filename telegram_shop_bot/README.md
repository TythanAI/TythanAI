# 🛒 Telegram-бот-магазин

Бот для автоматической продажи цифровых товаров прямо в Telegram: покупатель
выбирает товар, оплачивает **звёздами Telegram (Telegram Stars ⭐)** и тут же
получает товар. Всё внутри Telegram — никаких сайтов, карт и внешних платёжек
подключать не нужно.

> ⚠️ **Важно.** Перепродажа Telegram-аккаунтов нарушает
> [Условия использования Telegram](https://telegram.org/tos) — такой бот и
> аккаунт продавца могут заблокировать. Технически это универсальный магазин
> цифровых товаров: что именно загружать в каталог (ключи, подписки, файлы,
> данные для доступа и т. п.) — решаешь ты, и ответственность за это тоже на тебе.

---

## Что умеет бот

**Для покупателя:**
- 🛍 Каталог товаров с ценами и остатком
- 🛒 Покупка в один клик, оплата звёздами Telegram
- 📦 Мгновенная автоматическая выдача товара после оплаты
- 🧾 История покупок («Мои покупки»)
- 💬 Кнопка поддержки

**Для тебя (администратора):**
- ➕ Добавление товаров (мастер из 4 шагов)
- 📦 Загрузка «наличия» — по одной единице товара на строку
- 📋 Список товаров: скрыть/показать, удалить
- 📊 Статистика: продажи, выручка, пользователи, остатки
- 🔔 Уведомление о каждой продаже

Если наличие закончилось ровно в момент оплаты — бот **сам возвращает звёзды**
покупателю. Одну и ту же единицу товара нельзя продать дважды.

## ⚙️ Как всё подключить — по шагам

Ничего сложного, просто делай по порядку. Займёт минут 10.

### Шаг 1. Создай бота и получи токен

1. Открой в Telegram [@BotFather](https://t.me/BotFather).
2. Отправь команду `/newbot`.
3. Придумай имя бота (любое) и username (должен заканчиваться на `bot`, например `my_shop_bot`).
4. BotFather пришлёт **токен** — длинную строку вида
   `123456789:AAExxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`. Скопируй её — понадобится дальше.

### Шаг 2. Узнай свой Telegram ID

1. Открой [@userinfobot](https://t.me/userinfobot) и нажми **Start**.
2. Он пришлёт `Id: 123456789` — это твой ID. Скопируй число.
   Только этот ID будет видеть админ-панель.

### Шаг 3. Установи Python и зависимости

Нужен **Python 3.10+**. Проверь: `python --version`.

```bash
cd telegram_shop_bot
pip install -r requirements.txt
```

### Шаг 4. Настрой файл `.env`

Скопируй шаблон и открой его в любом редакторе:

```bash
cp .env.example .env
```

Заполни две обязательные строки:

```dotenv
BOT_TOKEN=сюда_вставь_токен_из_шага_1
ADMIN_IDS=сюда_вставь_свой_ID_из_шага_2
SHOP_NAME=Название твоего магазина
SUPPORT_CONTACT=@твой_username
```

`CURRENCY=XTR` менять не нужно — это оплата звёздами, ничего подключать не надо.

### Шаг 5. Запусти бота

```bash
python run.py
```

Если увидел в консоли `Бот @твой_бот запущен` — всё готово! Открывай своего бота
в Telegram и жми **Start**.

> Пока запущено это окно — бот работает. Закроешь — бот остановится. Чтобы он
> работал круглосуточно, смотри раздел **«Запуск 24/7»** ниже.

### Шаг 6. Наполни магазин (как админ)

1. Напиши боту `/start` → внизу появится кнопка **🛠 Админка** (её видишь только ты).
2. **➕ Добавить товар** → введи название, описание, категорию и цену **в звёздах**.
3. **📦 Добавить наличие** → выбери товар → пришли единицы товара,
   **по одной на строку**. Например:
   ```
   login1:pass1
   login2:pass2
   login3:pass3
   ```
   Каждая строка — отдельный товар, который бот выдаст очередному покупателю.
4. Готово — товар появился в каталоге у покупателей.

### Шаг 7. Как покупают клиенты

Клиент открывает бота → **🛍 Каталог** → выбирает товар → **🛒 Купить** →
подтверждает оплату звёздами → сразу получает товар. Ты получаешь уведомление 🔔.

## 💰 Про оплату звёздами (Telegram Stars)

- Покупатели платят **звёздами** — встроенной валютой Telegram. Звёзды они
  покупают прямо в Telegram (через Apple/Google/картой).
- Тебе как продавцу **не нужно** подключать банк, карту или платёжного провайдера.
- Полученные звёзды Telegram позволяет выводить в реальные деньги (через
  @BotFather → *Bot Settings*, при выполнении условий Telegram) или тратить внутри.
- Цену товара ты задаёшь в звёздах (целое число, минимум 1).

Хочешь принимать обычные деньги (рубли/карты) вместо звёзд? Это возможно через
внешнего провайдера (Telegram Payments): нужно получить `PROVIDER_TOKEN` у
[@BotFather](https://t.me/BotFather) и вписать его в `.env`, а `CURRENCY`
поменять на нужную валюту. Но для цифровых товаров Telegram рекомендует именно
звёзды — это проще и без посредников.

## 🖥 Запуск 24/7 (чтобы бот не выключался)

Чтобы бот работал постоянно, его запускают на сервере (VPS). Два простых способа.

### Вариант А. systemd (обычный Linux-сервер)

В комплекте есть готовый файл `deploy/shop-bot.service`. Отредактируй в нём пути
и пользователя, затем:

```bash
sudo cp deploy/shop-bot.service /etc/systemd/system/shop-bot.service
sudo systemctl daemon-reload
sudo systemctl enable --now shop-bot
sudo systemctl status shop-bot      # проверить, что работает
journalctl -u shop-bot -f           # смотреть логи
```

### Вариант Б. Docker

В комплекте есть `Dockerfile`:

```bash
docker build -t shop-bot .
docker run -d --name shop-bot --restart=always \
  --env-file .env -v "$PWD/data:/app/data" shop-bot
```

`-v .../data` сохраняет базу с товарами и заказами между перезапусками.

> Быстрый способ без своего сервера: залей папку на любой хостинг с поддержкой
> Python/Docker (например, Railway, Render, Fly.io), пропиши переменные из `.env`
> в настройках проекта и запусти `python run.py`.

## ❓ Если что-то не работает

| Проблема | Решение |
|---|---|
| `BOT_TOKEN не задан` | Не создан `.env` или пустой токен. Повтори шаги 1 и 4. |
| `ADMIN_IDS не задан` | Впиши свой числовой ID в `.env` (шаг 2). |
| Не вижу кнопку «Админка» | Твой ID в `ADMIN_IDS` не совпадает с аккаунтом, из которого пишешь боту. |
| Бот не отвечает | Проверь, что окно с `python run.py` запущено и нет ошибок в консоли. |
| `Unauthorized` при запуске | Неверный токен. Возьми новый у @BotFather (`/token`). |
| Оплата не проходит | Убедись, что у покупателя есть звёзды; для теста купи товар сам. |

## 🗂 Как устроен код

```
telegram_shop_bot/
├── run.py               # запуск бота
├── .env.example         # шаблон настроек
├── requirements.txt     # зависимости
├── Dockerfile           # для запуска в Docker
├── deploy/
│   └── shop-bot.service # systemd-юнит для запуска 24/7
├── bot/
│   ├── config.py        # чтение настроек из .env
│   ├── database.py      # база данных (SQLite): товары, наличие, заказы
│   ├── texts.py         # ВСЕ тексты бота — меняй их тут
│   ├── keyboards.py     # кнопки
│   ├── states.py        # шаги мастеров админки
│   ├── filters.py       # проверка «это админ?»
│   ├── utils.py         # вспомогательное
│   └── handlers/        # логика: user.py, admin.py, payments.py
└── tests/               # тесты (pytest)
```

Хочешь поменять тексты/эмодзи — правь `bot/texts.py`, код трогать не нужно.

Технологии: [aiogram 3](https://docs.aiogram.dev) + SQLite. База данных
создаётся автоматически в `data/shop.db` при первом запуске.

## 🔒 Безопасность

- Файлы `.env` и `data/*.db` **не** попадают в git (см. `.gitignore`) — там токен
  и данные заказов. Никому не показывай токен бота.
- Админ-панель доступна только ID из `ADMIN_IDS`. Все остальные её даже не видят.
